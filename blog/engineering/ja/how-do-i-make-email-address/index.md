# 良いメールアドレス
メールには、内容や形式だけでなく、アドレスにもマナーがあります。覚えやすい電話番号があるように、何かを依頼したいときに、頭の中で最も簡単に、真っ先に思い浮かぶメールアドレスであれば、ビジネスにおいて、あるいはヘッドハンターを通じて企業と繋がる際に、より多くの成果をもたらすことができるでしょう。

私たちのメールアドレスは、たいていIDの後にNaverやDaumのようなドメインが続きます。問題は、私たちがまだ幼い頃に「Naver Junior」のようなサービスを利用するために、これらのドメインに登録したことです。そのため、多くの場合、滑稽な、あるいはキャラクター名や可愛らしいIDが使われています。まるで、昔の「Buddy Buddy」（韓国で人気だったメッセンジャーサービス）のIDを思い出した時の恥ずかしさに似ています。この恥ずかしさは、当事者だけのものではありません。そのIDで連絡を受ける相手はどのように感じるでしょうか。

海外の大学や企業、あるいは外国人とのコミュニケーションを希望するのであれば、上記のような過去のアドレスの使用は避けるべきです。長年使い続けてきたメールアドレスのため、新しいメールアドレスを作成するのが不便であれば、メール転送機能の利用をお勧めします。

# おすすめのメールアドレス
一般的に、以下のルールに従うことを推奨します。最も重要なのは、覚えやすいメールアドレスであることです。
各ルールで使用する単語は、頭文字や略語でも構いませんが、できるだけ完全な単語を使用する方が記憶に残りやすいでしょう。

-   First Name (名)
-   First Name (名) + Last Name (姓)
-   Generic (汎用): メールの受信目的に合った名前

例えば、以下のような形です。

-   aaron@domain.com
-   aaronryu@domain.com
-   contact@domain.com

もしGmailをご利用であれば、以下のような便利な機能が提供されているので、活用することをお勧めします。

-   **ドット (.):** 自由にドットを挿入して可読性を高めることができます。ドットを付けない本来のメールアドレスに自動的に転送されます。
    -   例: aaron.ryu@domain.com
-   **プラス (+):** 特定のフィルターを適用した状態でメールを受信できます。これもまた、本来のメールアドレスに自動的に転送されます。例えば、履歴書に記載したメールアドレスが `aaron.ryu+recruit@domain.com` であれば、「recruit」フィルターのルールに従って振り分けられます。
    -   例: aaron.ryu+recruit@domain.com

# 重複するメールアドレスへの対処法
個人的にGmailでメールアドレスを作成しようとした際、希望するアドレスが既に使われていることに気づきました。
そのような場合、各IDやメール入力フォームに数字を付け加えたり、姓と名の順序を入れ替えたりすることが提案されますが、これらを使用しないことをお勧めします。外国人は名前の順序を「名 + 姓」として使用するため、その規則に従うのが良いでしょう。そうでなければ、外国人があなたを「姓」で呼んでしまうかもしれません。
また、生年を記載すると、特に年齢に関して東洋ほど敏感ではない外国人にとって、その人を思い出すたびに何年に生まれたかまで記憶しなければならないという不必要な情報を提供することになります。

このような場合、どうすれば良いでしょうか。
簡単な方法としては、職務を明記することが挙げられます。少し手間はかかりますが、独自のドメインを作成するのも良いでしょう。

-   aaron.ryu.dev@domain.com または aaronryudev@domain.com
-   aaronryu@aaronryu.com

IDだけでなく、ドメインにも自分の名前が重複して入ることでプロフェッショナルでないと感じるのではないかと心配されるかもしれませんが、その心配は不要です。[むしろ、自分自身のドメインを持っていること、そしてメールのやり取りで相手と直接話しているような印象を与えることができるでしょう。](https://workplace.stackexchange.com/questions/127195/does-repeating-name-in-email-address-look-unprofessional)
